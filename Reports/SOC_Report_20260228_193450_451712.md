### **3. Immediate Actions (SOC Analyst Steps)**
#### **A. Verify Automated Response**
- **Check if the brute force attempt was mitigated:**
  - Was the account locked? (Check Active Directory/LDAP logs)
  - Was the source IP blocked? (Check firewall/EDR logs)
  - Were failed login attempts logged? (Check SIEM for event correlation)

#### **B. Investigate the Attack**
- **Identify the Target:**
  - Which user/service account was targeted?
  - Is this a privileged account (admin, service account, etc.)?
- **Identify the Attacker:**
  - Source IP (geolocation, VPN/proxy usage, known malicious IPs?)
  - User-Agent (if web-based attack)
  - Timeframe of the attack (burst or prolonged?)
- **Check for Successful Logins:**
  - Were any attempts successful? (Check for "Accepted" logins post-failure)
  - If yes → **Escalate to Incident Response (IR) for compromise assessment.**

#### **C. Hunt for Related Activity**
- **Lateral Movement:** Are there signs of the attacker moving to other systems?
- **Data Exfiltration:** Unusual outbound traffic from the targeted system?
- **Persistence:** New accounts, scheduled tasks, or backdoors created?

#### **D. Strengthen Defenses (If Needed)**
- **If brute force is recurring:**
  - Enforce **account lockout policies** (e.g., 5 failed attempts → 30-min lockout).
  - Implement **MFA** (if not already enabled).
  - Enable **rate-limiting** on login endpoints.
  - Block known malicious IPs via **firewall/EDR**.
- **If the attack was successful:**
  - **Isolate the affected system.**
  - **Rotate credentials** for the compromised account.
  - **Check for data breaches** (e.g., unauthorized access to sensitive files).
