1. **Containment:**
   - **Isolate the compromised system(s)** from the network (disable NIC, VLAN segmentation, or firewall rules).

   - **Revoke active sessions** (e.g., `klist purge` for Kerberos, `net use /delete` for SMB).

2. **Eradication:**
   - **Rotate all credentials** for accounts accessed by the compromised user (including service accounts).
   - **Enable MFA** for all privileged accounts if not already enforced.
   - **Block the source IP(s)** at the perimeter firewall/IDS/IPS.

3. **Evidence Preservation:**
   - **Collect forensic artifacts:**
     - Memory dump (`winpmem`, `AVML`).

   - **Document the timeline** of events (attack start/end, successful logins, lateral movement).

### **Short-Term Actions (2–24 Hours)**

   - **Review logs** for additional compromised accounts or systems.

   - **Analyze lateral movement paths** (e.g., RDP, SMB, PsExec).

2. **Remediation:**
   - **Patch exposed services** (e.g., disable RDP if unnecessary, enforce NLA).
   - **Implement account lockout policies** (e.g., 5 failed attempts = 30-minute lockout).
   - **Deploy deception techniques** (e.g., honey accounts, canary tokens).

3. **Communication:**
