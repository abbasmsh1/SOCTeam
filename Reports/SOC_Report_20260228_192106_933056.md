### **3. Immediate Actions (Tier 1 SOC Analyst)**
#### **A. Verify the Attack**
- **Check logs** for:
  - Multiple failed login attempts (same/different usernames).
  - Source IP(s) (single IP or distributed?).
  - Targeted service (RDP, SSH, VPN, web app, etc.).
- **Example queries:**
  ```sql
  -- Windows (Event ID 4625 - Failed Logon)
  SELECT * FROM WinEventLog WHERE EventID=4625 AND TimeGenerated > NOW()-1H;

  -- Linux (auth.log)
  grep "Failed password" /var/log/auth.log | tail -20;

  -- Web App (e.g., Apache/Nginx)
  cat /var/log/apache2/access.log | grep "POST /login" | grep "401";
  ```

#### **B. Assess Impact**
- **Was the attack successful?**
  - Check for **Event ID 4624** (Windows) or **accepted logins** (Linux) from the same IP.
  - Look for **unusual account activity** post-attack (e.g., lateral movement, data access).
- **Are critical accounts targeted?**
  - Admin, service, or high-privilege accounts?

#### **C. Validate Automated Response**
- **If IP blocking was triggered:**
  - Confirm the IP is now blocked (firewall/EDR logs).
  - Check if the attack has **stopped** or **shifted to a new IP**.
- **If account lockout was triggered:**
  - Verify the account is locked (`net user <username>` or AD).
  - Check for **password spray patterns** (same password across multiple accounts).

---

### **4. Escalation & Mitigation (If Needed)**
#### **A. If Attack is Ongoing/Unmitigated:**
- **Manual block:** Add the IP to firewall/EDR blacklist.
- **Rate limiting:** Configure login attempt thresholds (e.g., fail2ban, Azure AD Smart Lockout).
- **Password policy:** Enforce **MFA** and **stronger passwords** for targeted accounts.

#### **B. If Attack Succeeded:**
- **Escalate to Tier 2/3:**
  - **Reset compromised credentials** (force password change + revoke sessions).
  - **Hunt for lateral movement** (check for unusual logins, process execution, or data exfil).
  - **Check for persistence** (new accounts, scheduled tasks, or backdoors).

#### **C. If False Positive (Unlikely but Possible):**
- **Check for:**
  - Legitimate automated tools (e.g., vulnerability scanners, backup scripts).
  - Misconfigured services (e.g., default credentials on a test server).
- **Tune detection rules** if needed (e.g., whitelist known IPs).

---

### **5. Long-Term Recommendations**
🔹 **Enhance brute force defenses:**
  - Deploy **fail2ban** (Linux) or **Windows Defender Exploit Guard**.
  - Enable **MFA** for all remote access (RDP, VPN, email).
  - Use **honeypot accounts** to detect attacks early.

🔹 **Improve logging & monitoring:**
  - Forward logs to **SIEM** (Splunk, ELK, Sentinel) for correlation.
  - Set up **alerts for brute force patterns** (e.g., 5+ failed logins in 1 minute).

🔹 **Threat hunting:**
  - Search for **historical brute force attempts** (may indicate a larger campaign).
  - Check **Tor exit nodes/VPN IPs** in threat intelligence feeds.

---

### **6. Next Steps**
- **Update ticket** with findings (successful/failed, actions taken).
- **Close alert** if mitigated or **escalate** if breach confirmed.
- **Document IOCs** (IPs, usernames, timestamps) for future reference.
