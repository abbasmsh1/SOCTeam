---

### **Manual Investigation Steps**
#### **1. Verify the Brute Force Attack**
- **Check logs for:**
  - **Failed login attempts** (e.g., Windows Event ID **4625**, Linux `/var/log/auth.log`).
  - **Source IP reputation** (e.g., via [AbuseIPDB](https://www.abuseipdb.com/), [VirusTotal](https://www.virustotal.com/)).
  - **Targeted accounts** (Are they privileged? Service accounts? Default accounts like `admin`?).
- **Questions to answer:**
  - Is this a **known bad IP** (e.g., Tor exit node, VPN, cloud provider)?
  - Are the **credentials being tested** (e.g., `admin:admin`, `root:password`)?
  - Is there a **pattern** (e.g., sequential attempts, password spraying)?

#### **2. Assess Impact & Compromise Indicators**
- **Successful logins after brute force?**
  - Check for **Event ID 4624** (Windows) or `Accepted password` (Linux).
  - If yes → **Assume breach** and initiate **incident response (IR)**.
- **Lateral movement or post-exploitation?**
  - Look for **unusual processes** (e.g., `mimikatz`, `PsExec`, `PowerShell Empire`).
  - Check **network connections** (e.g., `netstat`, `Zeek`, `Suricata` logs).
- **Data exfiltration?**
  - Monitor for **large outbound transfers** (e.g., `SMB`, `FTP`, `HTTP POST` to unknown domains).

#### **3. Containment & Eradication**
- **If no compromise is found:**
  - **Block the source IP** (firewall, WAF, or EDR).
  - **Enforce account lockout policies** (if not already automated).
  - **Rotate credentials** for targeted accounts (especially if passwords were weak).
- **If compromise is confirmed:**
  - **Isolate affected systems** (network segmentation, disable RDP/SSH).
  - **Reset all credentials** (not just the targeted ones).
  - **Hunt for persistence** (e.g., scheduled tasks, cron jobs, startup scripts).
  - **Engage IR team** if scope is large (e.g., multiple systems, data theft).

#### **4. Long-Term Mitigations**
- **Prevent future brute force:**
  - **Enable MFA** (especially for remote access).
  - **Rate-limit login attempts** (e.g., fail2ban, Azure AD Smart Lockout).
  - **Disable legacy protocols** (e.g., SMBv1, Telnet, FTP).
  - **Deploy deception tech** (e.g., honey accounts, canary tokens).
- **Improve detection:**
  - **Tune IDS/IPS rules** to reduce false positives.
  - **Set up alerts for brute force patterns** (e.g., 5+ failed logins in 1 minute).
  - **Integrate UEBA** (User Entity Behavior Analytics) for anomalous login detection.
